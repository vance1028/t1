## 1. 产品概述
城市地下综合管廊运维控制台——为管廊运营方提供一张图的实时态势感知、分级告警、安全联动和巡检调度能力，取代人工下井巡检的滞后模式，实现"地面盯住整条管廊"。

- 解决管廊监控依赖人工巡检、告警响应慢、联动动作无追溯的痛点
- 目标用户：管廊运营调度员、巡检员、安全管理员

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 调度员 | 管理员分配账号 | 查看态势、处理告警、派发工单、配置阈值 |
| 巡检员 | 管理员分配账号 | 接收工单、打卡上报、拍照挂隐患单 |
| 管理员 | 系统初始账号 | 全部权限 + 用户管理 + 阈值配置 |

### 2.2 功能模块
1. **态势总览页**：管廊舱室拓扑图 + 实时环境数据 + 告警高亮
2. **告警中心页**：分级告警列表 + 告警详情 + 联动日志追溯
3. **巡检调度页**：工单管理 + 派工 + 打卡上报 + 防冲突校验
4. **阈值配置页**：各传感器阈值设置 + 联动规则配置

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 态势总览 | 舱室拓扑图 | 按管廊结构绘制舱段/舱室节点，节点实时显示温湿度、氧含量、燃气浓度、积水液位、防火门状态；越限舱室变红闪烁 |
| 态势总览 | 环境数据面板 | 选中舱室后展示详细传感器读数曲线和当前值 |
| 态势总览 | 告警摘要条 | 页面顶部横幅展示当前活跃告警数量和最高级别 |
| 告警中心 | 告警列表 | 分级（一级提示/二级联动/三级紧急）展示所有告警，支持按级别/舱室/时间筛选 |
| 告警中心 | 联动日志 | 记录每次联动触发的动作（排风、禁入、启泵）、触发时间、触发值、执行结果 |
| 巡检调度 | 工单列表 | 展示所有工单及状态（待派/进行中/已完成），支持筛选 |
| 巡检调度 | 派工表单 | 选择巡检员、舱段、检查项、时间段，提交前校验冲突 |
| 巡检调度 | 打卡上报 | 巡检员按点位逐项打卡，发现隐患可拍照上传 |
| 阈值配置 | 阈值表 | 各传感器类型的一级/二级阈值配置 |
| 阈值配置 | 联动规则 | 燃气超限→排风+禁入、积水超限→启泵等规则展示与启停 |

## 3. 核心流程

### 3.1 实时监控与告警流程
传感器模拟器每秒推送数据 → WebSocket 广播至前端 → 前端更新拓扑图节点 → 后端同步判定阈值越限 → 越限时生成告警记录 → 触发联动动作（排风/禁入/启泵）→ 联动日志落库 → 告警状态推送前端

```mermaid
flowchart TD
    "传感器模拟器" --> "每秒推送数据"
    "每秒推送数据" --> "后端WebSocket广播"
    "后端WebSocket广播" --> "前端拓扑图实时更新"
    "每秒推送数据" --> "后端阈值判定引擎"
    "后端阈值判定引擎" -->|"越限"| "生成分级告警"
    "后端阈值判定引擎" -->|"超二级阈值"| "触发安全联动"
    "触发安全联动" --> "排风/禁入/启泵"
    "触发安全联动" --> "联动日志落库"
    "生成分级告警" --> "告警推送至前端"
```

### 3.2 巡检派工防冲突流程
调度员派工 → 系统校验：同一舱段同一时段是否有其他工单 → 有动火作业时该舱段是否锁定 → 校验通过则创建工单 → 校验不通过则提示冲突原因

```mermaid
flowchart TD
    "调度员提交派工" --> "校验舱段时段冲突"
    "校验舱段时段冲突" -->|"无冲突"| "校验危险作业锁定"
    "校验舱段时段冲突" -->|"有冲突"| "返回冲突提示"
    "校验危险作业锁定" -->|"未锁定"| "创建工单"
    "校验危险作业锁定" -->|"已锁定"| "返回锁定提示"
```

### 3.3 安全联动逻辑
| 触发条件 | 联动动作 |
|----------|----------|
| 燃气舱燃气浓度 > 一级阈值 | 提示告警（一级） |
| 燃气舱燃气浓度 > 二级阈值 | 联动启动排风 + 禁止人员进入 + 二级告警 |
| 积水液位 > 高位阈值 | 联动启动排水泵 + 告警 |

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深蓝灰底色（#0f1923）+ 青蓝强调色（#00d4ff），营造工业监控的科技感
- **告警色**：一级黄色（#f0b429）、二级橙色（#ff6b35）、三级红色（#e53e3e）
- **正常状态**：绿色（#38b2ac）
- **按钮风格**：圆角矩形，hover 时发光效果
- **字体**：数字用 JetBrains Mono 等宽字体（传感器读数），正文用 Noto Sans SC
- **布局**：左侧导航栏 + 右侧内容区，拓扑图居中占主区域
- **图标**：Lucide Icons

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 态势总览 | 舱室拓扑图 | 深色底、横向舱段排列、每个舱室卡片显示传感器数值、越限时红色边框闪烁动画 |
| 态势总览 | 告警摘要条 | 顶部固定横幅，按级别显示告警计数徽章 |
| 告警中心 | 告警列表 | 表格式，级别用色标，支持筛选器 |
| 告警中心 | 联动日志 | 时间线形式，每条日志含触发值和执行结果 |
| 巡检调度 | 工单列表 | 卡片式列表，状态用色标区分 |
| 巡检调度 | 派工表单 | 模态弹窗，下拉选择巡检员和舱段 |
| 阈值配置 | 阈值表 | 可编辑表格，数值型输入 |

### 4.3 响应式
桌面端优先（1920×1080 为基准），支持 1366×768 最小分辨率。移动端巡检员打卡页面做基础适配。

## 5. 种子数据规划

### 5.1 管廊结构
- 4 个舱段（A1-A4），每段含：电力舱、燃气舱、综合舱（共 12 个舱室）
- 每个舱室布设：温湿度传感器、氧含量传感器、燃气浓度传感器、积水液位传感器、防火门状态传感器

### 5.2 初始传感器读数
- 大部分正常值：温度 20-25°C、湿度 40-60%、氧含量 19-21%、燃气浓度 <5、积水液位 <10cm、防火门关闭
- 故意异常：A2-燃气舱燃气浓度设为 28（超一级阈值）、A3-综合舱积水液位设为 35cm（超警戒）、A1-电力舱温度设为 38°C（接近阈值）

### 5.3 巡检员和工单
- 3 名巡检员
- 待派工单 1 条、进行中 1 条、已完成 1 条

### 5.4 告警和联动日志
- 3 条历史告警记录
- 2 条联动日志样例

### 5.5 阈值配置默认值
| 传感器类型 | 一级阈值 | 二级阈值 | 单位 |
|-----------|----------|----------|------|
| 温度 | 35 | 45 | °C |
| 湿度 | 70 | 85 | %RH |
| 氧含量 | 19.5 | 18 | %VOL（低于一级即告警） |
| 燃气浓度 | 20 | 40 | LEL% |
| 积水液位 | 20 | 35 | cm |
